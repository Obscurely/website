## Introduction to Next.js and TypeScript

Next.js is a React framework that enables functionality such as server-side rendering and static site generation. TypeScript is a strongly typed programming language that builds on JavaScript. When combined, they provide a powerful development experience.

### Why Use Next.js?

Next.js provides several benefits out of the box:

- **Server-side rendering (SSR)**: Improves performance and SEO
- **Static site generation (SSG)**: Pre-renders pages at build time
- **API Routes**: Create API endpoints easily
- **File-based routing**: Simplifies navigation
- **Built-in CSS and Sass support**: Streamlines styling

<Callout>
  Next.js 13 introduced the App Router, a new paradigm for building Next.js applications with enhanced features like React Server Components.
</Callout>

## Setting Up Your Project

### Creating Your First Component

Let's create a simple component in TypeScript:

```typescript
// src/components/Button.tsx
import React from 'react';

interface ButtonProps {
  text: string;
  onClick?: () => void;
  variant?: 'primary' | 'secondary';
}

export function Button({ text, onClick, variant = 'primary' }: ButtonProps) {
  const baseStyles = "px-4 py-2 rounded font-medium";
  const variantStyles = variant === 'primary' 
    ? "bg-blue-500 text-white hover:bg-blue-600" 
    : "bg-gray-200 text-gray-800 hover:bg-gray-300";
  
  return (
    <button 
      className={`${baseStyles} ${variantStyles}`}
      onClick={onClick}
    >
      {text}
    </button>
  );
}
```

## TypeScript Benefits

TypeScript provides several advantages when working with React and Next.js:

1. **Type Safety**: Catch errors during development instead of runtime
2. **Better IDE Support**: Enhanced autocomplete and documentation
3. **Improved Refactoring**: Safely rename and move code
4. **Self-Documenting Code**: Types serve as documentation

<Callout type="warning">
  While TypeScript adds many benefits, it does have a learning curve. Start with basic type annotations and gradually adopt more advanced features.
</Callout>

// Generic components
function List<T>({ items, renderItem }: {

  renderItem: (item: T) => React.ReactNode;

      {items.map((item, index) => (
